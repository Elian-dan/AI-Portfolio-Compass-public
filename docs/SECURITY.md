# 数据安全边界

## 第一版边界

- 只读取富途 OpenD 数据，不执行交易动作。
- 不保存交易密码。
- 不调用 SDK 的交易解锁能力。
- 不上传本地数据库。
- 错误信息应截断展示，避免在日志中暴露完整账户和资产数据。

## 数据保留

本地 SQLite 保存：

- 账户信息
- 账户资产快照
- 持仓快照
- 历史成交
- 自选股
- 行情摘要
- 决策卡
- AI 分析记录
- 提醒
- 同步任务
- 用户画像版本
- 用户手动仓位修正

用户在设置页执行删除本地数据时，会清空上述本地数据；主动导出的文件不在自动删除范围内。

## AI 分析

当前实现支持 DeepSeek API，也保留本地规则降级：

- API Key 只从环境变量 `DEEPSEEK_API_KEY` 读取，不入库。
- 发送给 DeepSeek 的内容限定为单个标的分析上下文，包括持仓摘要、仓位分类、决策卡、画像摘要和数据新鲜度。
- 不发送本地 SQLite 文件、交易密码、OpenD 解锁信息。
- 不实现下单、撤单、改单或交易解锁。
- 外部模型输出会经过强指令词过滤，避免出现“立即买入 / 立即卖出 / 必涨 / 必跌 / 稳赚 / 满仓”等文案。
- 未配置 Key 或外部调用失败时，页面会展示本地降级分析状态。
